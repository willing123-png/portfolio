# 项目详细页和响应式规则

## 调整目标

把项目详细页抽成一个通用案例组件，适合展示多张图片从上到下排列的作品集案例页。当前实际素材为 PNG；后续如果需要，也可以换成 JPG 或 WebP。马蜂窝项目先作为第一个使用对象，后续 Youware、哈皮等详情页也可以复用同一套结构。

## 通用组件方案

新建一个通用组件，例如：

```text
portfolio/src/components/CaseImageGallery.astro
```

组件负责：

- 顶部项目信息：项目编号、标题、副标题、标签或简短说明。
- 返回入口：返回 `/#projects`。
- 图片流：多张图片垂直排列。
- 响应式宽度规则：页面区域铺满屏幕；移动端内容容器铺满屏幕并保留统一内边距；桌面端内容容器适配首页计划。

## 响应式宽度规则

项目详情页的内容宽度分移动端和桌面端处理。

页面区域：

```css
width: 100%;
```

移动端内容容器：

```text
w-full max-w-none px-3
```

说明：

- 移动端详情页内容容器铺满屏幕。
- 移动端详情页内容容器内部统一保留 `12px` 左右 padding。
- 移动端详情页中的图片和文字使用同一套内容内边距。

桌面端内容容器：

```text
md:mx-auto md:w-full md:max-w-[1300px] md:px-16
```

说明：

- 桌面端详情页内容容器限制最大宽度，并水平居中。
- 桌面端内容容器最大宽度与首页一致，暂定 `1300px`。
- 暂不使用旧的桌面端 `md:w-[60vw] md:max-w-[1120px]` 作为项目详情页内容宽度规则。

图片流：

```text
图片容器跟随内容容器宽度
图片自身使用 w-full h-auto
```

说明：

- 图片从上到下排列在内容容器内。
- 移动端图片宽度跟随带 `12px` 内边距的内容容器。
- 桌面端图片宽度跟随居中且限制最大宽度的内容容器。
- 图片高度保持自动，避免变形。

## 移动端铺满屏幕的对象说明

移动端“铺满屏幕”指的是：

- 页面区域铺满屏幕。
- section 背景铺满屏幕。
- 项目详情页内容容器铺满屏幕。

移动端“铺满屏幕”不指：

- 图片和文字贴到屏幕边缘。
- 首页内容容器也取消左右 padding。
- 桌面端内容容器取消最大宽度限制。

## 马蜂窝页面改造

`mafengwo.astro` 不再写完整布局，只传配置给通用组件：

```astro
<CaseImageGallery
  projectNo="PROJECT 02"
  title="马蜂窝 · 营销活动设计"
  subtitle="校企合作项目"
  images={images}
/>
```

## 图片展示细节

- 图片从上到下单列排列。
- 移动端图片之间使用 `gap-3`，即 `12px`。
- 桌面端图片之间使用 `md:gap-8`，即 `32px`。
- 每张图片使用 `loading="lazy"` 和 `decoding="async"`。
- 每张图片保留 `alt` 文案，避免纯图片页面缺少可访问性信息。
- 如果图片里包含重要文字，后续可考虑在页面里额外保留简短真实文本。

## 后续复用

后续项目页只需要替换：

- 项目编号
- 项目标题
- 项目副标题
- 图片数组
- 可选标签或说明

这样可以保持所有案例页的阅读体验一致，也能减少后续维护成本。
