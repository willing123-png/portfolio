# 首页响应式调整计划

## 目标

调整首页在桌面端和移动端的响应式规则：

- 页面区域始终铺满屏幕。
- 背景始终全屏铺满，不受内容最大宽度限制。
- 内容区域在桌面端居中，并限制最大宽度。
- 内容区域在移动端跟随屏幕宽度变化。
- 桌面端字号固定。
- 移动端字号固定。
- 不使用 `vw` 或 `clamp()` 控制字号。

## 调整原则

### 页面画布

- `html`、`body` 保持无默认外边距。
- `main` 保持 `width: 100%`。
- 每个 section 保持 `width: 100%`。
- 页面背景和 section 背景铺满整个浏览器宽度。

这里的“铺满屏幕”指页面区域、section 区域和背景区域，不指所有内容都贴到屏幕边缘。

### 内容容器

桌面端：

- 内容容器 `width: 100%`。
- 内容容器设置 `max-width`。
- 内容容器水平居中。
- 内容左右保留固定 padding。

移动端：

- 内容容器 `width: 100%`。
- 内容容器不设置固定宽度。
- 内容左右保留固定 padding。
- 内容实际可用宽度随屏幕宽度变化。

### 字号

桌面端和移动端都使用固定字号。

不再使用：

```css
font-size: clamp(...);
font-size: 8vw;
font-size: 13vw;
```

改为按断点设置固定值。

本次执行采用以下固定字号：

| 对象 | `< 768px` | `>= 768px` | `>= 1024px` | `>= 1280px` |
|---|---:|---:|---:|---:|
| `PORTFOLIO` | `44px` | `96px` | `132px` | `180px` |
| `WORK` | `40px` | `76px` | `102px` | `140px` |
| `PROJECT` / `ARTICLE` | `48px` | `120px` | `120px` | `120px` |

示例：

```css
.hero-title {
  font-size: 44px;
}

@media (min-width: 768px) {
  .hero-title {
    font-size: 96px;
  }
}
```

## 具体调整步骤

### 1. 统一内容容器规则

保留类似结构：

```js
const wrap = "mx-auto w-full max-w-[1300px] px-6 md:px-16";
```

本次执行保留：

- 桌面端最大宽度使用 `1300px`。
- 移动端左右 padding 使用 `24px`。
- 桌面端左右 padding 使用 `64px`。

### 2. 将行内字号迁移到 CSS class

把 Hero 中的行内字号：

```jsx
style={{ fontFamily: 'var(--font-display)', fontSize: 'clamp(...)' }}
```

改为 class 控制，例如：

```jsx
className="hero-title font-black leading-[0.82] tracking-tight"
```

再在 CSS 中集中设置移动端和桌面端固定字号。

### 3. 固定 Hero 字号

为以下元素设置固定字号：

- `PORTFOLIO`
- `WORK`
- `↗`
- 联系信息

示例结构：

```css
.hero-title {
  font-size: 44px;
}

.hero-work {
  font-size: 40px;
}

@media (min-width: 768px) {
  .hero-title {
    font-size: 96px;
  }

  .hero-work {
    font-size: 76px;
  }
}

@media (min-width: 1024px) {
  .hero-title {
    font-size: 132px;
  }

  .hero-work {
    font-size: 102px;
  }
}

@media (min-width: 1280px) {
  .hero-title {
    font-size: 180px;
  }

  .hero-work {
    font-size: 140px;
  }
}
```

### 4. 固定区块标题字号

将 `PROJECT` 和 `ARTICLE` 从：

```css
font-size: clamp(48px, 8vw, 120px);
```

改为固定字号 class，例如：

```css
.section-display-title {
  font-size: 48px;
}

@media (min-width: 768px) {
  .section-display-title {
    font-size: 120px;
  }
}
```

### 5. 保持背景全屏

Hero 背景继续放在 section 内部的绝对定位层：

```css
position: absolute;
inset: 0;
width: 100%;
height: 100%;
```

背景不放进 `max-width` 内容容器。

文字内容继续放进居中的内容容器。

### 6. 保持移动端布局自适应

移动端自适应指布局宽度自适应，不指字号自适应。

移动端保持：

- 页面区域、section 区域和背景区域铺满屏幕。
- 内容容器宽度 `100%`。
- 内容容器保留左右 padding。
- 卡片宽度随容器变化。
- 图片宽度可按固定宽度或百分比设置。
- 项目和文章卡片默认单列。

### 7. 保持桌面端布局规则

桌面端保持：

- 页面区域全屏。
- 背景全屏。
- 内容区域最大宽度限制。
- 内容区域居中。
- 项目卡片按桌面规则多列显示。
- 字号不随浏览器继续放大或缩小。
